# 快速啟動指南

> 完整架構、Mock/API 切換、功能說明請看 [README.md](README.md)。
> 本檔只放「最快把專案跑起來」的步驟。

## 立即運行

```bash
# 1. 安裝依賴
flutter pub get

# 2. 以 Chrome 啟動開發
flutter run -d chrome
```

> Flutter Web 使用 CanvasKit 渲染（在 `web/index.html` 以 `flutter_bootstrap.js` 載入）。

## 登入

預設為 **Mock 模式**（本地假資料，不連後端），輸入**任意非空帳號 / 密碼**即可登入
（例如 `admin` / `123456`）。登入後導向 `/#/dashboard`。

## 畫面路由

- `/login` — 登入頁
- `/dashboard` — 後台主框架（左側選單 + 內容區）
  - 預設選中「基本資料作業」→ 顯示「使用者帳號」清單頁

## 接下來想做什麼

| 需求 | 動哪裡 |
|------|--------|
| 新增一個畫面 | 在 `lib/app/modules/<功能>/` 建 `controllers / bindings / views`，並在 `lib/app/routes/app_pages.dart` 註冊路由 |
| 連接正式後端 | 把 `lib/app/core/values/app_constants.dart` 的 `useMock` 改成 `false`（service/controller 不用動） |
| 新增一支 API | 在 service 用 dio 呼叫，並同步在 `lib/app/data/providers/mock_interceptor.dart` 補對應 mock route |
| 改樣式 / 配色 | `lib/app/core/theme/app_colors.dart`（不要在 widget 寫死 hex） |

## 常見問題

**Q: 如何登出？**
A: 點右上角帳號旁的頭像 →「登出」。

**Q: 為什麼首次載入較慢？**
A: CanvasKit 首次需下載約 2–3MB 的引擎檔，之後會被瀏覽器快取。

**Q: 改了 `pubspec.yaml`（套件 / 字型 / 資產）沒生效？**
A: 需重新 `flutter run`，hot reload 不會載入 pubspec 變更。
