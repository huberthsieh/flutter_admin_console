# Flutter Admin Console 開發步驟

> ⚠️ 此文件記錄的是早期 **go_router** 版本的開發歷程，僅供參考。
> 專案已改用 **GetX** 架構，最新技術說明與運行方式請看 [README.md](README.md)。

## 專案概述
使用 Flutter + CanvasKit + go_router 開發一個簡單的網站後台管理系統

## 功能需求
1. 登入功能
2. 後台介面佈局：
   - Header（頂部）
   - Sidebar 選單（左側）
   - 主要功能區域（右側）
3. 樣式參考 Flutter_bootstrap

## 開發步驟

### 1. 初始化專案
- 創建 Flutter 專案
- 配置 CanvasKit renderer
- 添加必要依賴

### 2. 依賴套件
```yaml
dependencies:
  flutter:
    sdk: flutter
  go_router: ^latest
  flutter_bootstrap: ^latest  # 用於參考樣式
```

### 3. 專案結構
```
lib/
├── main.dart
├── router/
│   └── app_router.dart
├── screens/
│   ├── login_screen.dart
│   └── dashboard/
│       ├── dashboard_screen.dart
│       ├── widgets/
│       │   ├── header.dart
│       │   ├── sidebar.dart
│       │   └── main_content.dart
├── services/
│   └── auth_service.dart
└── models/
    └── user.dart
```

### 4. 核心功能實作

#### 4.1 路由配置（go_router）
- 登入頁面路由
- 後台主頁路由
- 路由守衛（未登入重導向）

#### 4.2 登入頁面
- 表單輸入（帳號、密碼）
- 登入驗證
- 登入成功後導向後台

#### 4.3 後台佈局
- Header: 導航列、用戶資訊、登出按鈕
- Sidebar: 選單列表、可折疊
- MainContent: 主要內容區域

### 5. 樣式設計
- 參考 Flutter_bootstrap 的配色方案
- 響應式設計
- Material Design 組件

### 6. Web 配置
- 配置 CanvasKit renderer
- 修改 index.html

## 技術要點

### CanvasKit 配置
在 `web/index.html` 中配置：
```javascript
window.addEventListener('load', function(ev) {
  _flutter.loader.loadEntrypoint({
    serviceWorker: {
      serviceWorkerVersion: serviceWorkerVersion,
    },
    onEntrypointLoaded: function(engineInitializer) {
      engineInitializer.initializeEngine({
        renderer: "canvaskit",
      }).then(function(appRunner) {
        appRunner.runApp();
      });
    }
  });
});
```

### go_router 路由守衛
使用 `redirect` 回調函數檢查登入狀態
